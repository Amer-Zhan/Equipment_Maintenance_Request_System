# Short Project Explanation for Video Presentation

## Project Name
Equipment Maintenance and Workover Request Tracking System

## Project Purpose
The project is a database-oriented prototype of an internal information system for an oilfield service company. The system helps organize information about equipment, maintenance requests, request statuses, employees, and maintenance history.

## Technologies Used
- PostgreSQL for database storage
- SQL for creating tables, inserting sample data, and writing queries
- Python for connecting to the database and processing information
- HTML and CSS for the static interface prototype
- GitHub for source code storage
- diagrams.net / draw.io for ERD diagram preparation

## Main Database Tables
- departments
- employees
- equipment
- request_statuses
- workover_requests
- maintenance_logs

## Main Features
- Equipment register
- Maintenance and workover request tracking
- Request status control
- Request filtering by priority and status
- Maintenance history storage
- Dashboard summary using SQL and Python
- Static interface prototype for system visualization

## What I Learned
During this project, I practiced database design, SQL queries, PostgreSQL usage, Python database connection, data processing, and HTML/CSS interface prototyping. I also learned how to structure a technical project in a way that can be stored and presented through GitHub.
